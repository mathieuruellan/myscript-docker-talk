## What we do with Docker? - MyScript Cloud (online recognition) - MyScript ATK (certificate provider for mobile apps embedding our widgets) - Builds
## Who is using Docker? - developers - testers - production (AWS/EC2 environment)
## What do we need in production? - Backend - DB (mongodb) - SMTP relay (Postfix) - HAProxy (load balancing) - logger (rsyslog - elasticsearch/logstash/kibana) - Varnish - static content (nginx)
## Deliveries are Docker Images - Environment agnostic (thanks to environement production) - Build result are pushed in a local registry (it's also a docker container) - Production deployment on private S3 with **docker save** and **docker load** - Possibility to run a production-like environment with fig/docker-compose
## Best practices - One process per container - Pipe the process stdout/stderr to logger otherwise mount a /dev/log volume (deprecated with --log-driver since 1.6) - In Production, use **`docker run --net=host ...`** - Do not believe that it's safe
